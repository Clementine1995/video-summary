# 视频内容总结工具开发需求文档

## 1. 项目目标

开发一个面向讲座、访谈、课程类视频的内容总结工具。工具不以画面理解为核心，而是优先基于字幕和音频转写生成结构化学习笔记。

用户输入 YouTube、B站视频链接或本地视频/音频文件后，系统自动完成字幕获取、音频转写、文本清洗、分段总结、全局归纳和 Markdown 导出。

## 2. 适用场景

### 2.1 主要场景

- 讲座视频
- 访谈视频
- 在线课程
- 播客视频
- 会议录屏中以口述为主的内容

### 2.2 暂不重点支持

- 影视剧情解说
- 游戏实况
- 强依赖画面的教程
- 纯视觉演示类视频
- 需要逐帧理解的视频

## 3. 核心原则

1. 字幕优先：如果视频已有字幕，优先使用字幕，降低成本和耗时。
2. 转写兜底：没有字幕或字幕质量较差时，使用 Whisper/faster-whisper 进行音频转写。
3. 分块处理：长视频先切分，再分别总结，最后汇总，避免上下文过长。
4. 可追溯：摘要中的关键观点尽量保留时间戳，方便用户回看。
5. 可替换：下载、转写、LLM 总结模块应解耦，便于后续替换实现。

## 4. MVP 功能范围

### 4.1 输入能力

支持三类输入：

- YouTube 视频链接
- B站视频链接
- 本地视频或音频文件

验收标准：

- 输入 YouTube 链接后，系统能识别视频标题、时长和可用字幕状态。
- 输入 B站链接后，系统能识别视频标题、分 P 信息和基础元数据。
- 输入本地文件后，系统能识别文件格式、时长和音频轨道。
- 不支持的链接或文件格式必须返回清晰错误信息。

### 4.2 字幕获取

系统优先尝试获取官方字幕或自动字幕。

建议实现：

- YouTube：使用 youtube-transcript-api 或 yt-dlp 获取字幕。
- B站：优先尝试平台字幕接口；失败时进入音频转写流程。
- 本地文件：检测内嵌字幕；没有字幕则进入音频转写流程。

验收标准：

- 有字幕的 YouTube 视频可以直接生成原始字幕文本。
- 无字幕视频会自动进入转写队列。
- 字幕文本需包含时间戳、原文内容和语言信息。
- 字幕获取失败时，错误原因被记录到任务日志。

### 4.3 音频提取

当没有可用字幕时，系统从视频中提取音频。

建议实现：

- 使用 yt-dlp 下载在线视频音频。
- 使用 ffmpeg 从视频中抽取音频。
- 统一转换为 16kHz 或 24kHz 的 wav/mp3 格式，供 ASR 使用。

验收标准：

- YouTube/B站链接可以成功生成音频文件。
- 本地视频可以成功提取音频轨道。
- 无音频轨道的视频必须提示“未检测到可用音频”。
- 提取后的音频时长与原视频时长误差小于 2 秒。

### 4.4 语音转写

使用本地 ASR 模型将音频转为文字。

建议实现：

- 首选 faster-whisper。
- 可选 whisper.cpp，用于更轻量的本地部署。
- 支持中文、英文和中英混合内容。
- 输出带时间戳的分段文本。

验收标准：

- 10 分钟中文访谈视频可成功转写。
- 转写结果包含每段开始时间、结束时间和文本。
- 系统能识别中文、英文或混合语言。
- 转写失败时保留原始音频，方便重新处理。

### 4.5 文本清洗

对字幕或转写文本进行清洗，提升后续总结质量。

清洗内容包括：

- 合并过短字幕行
- 删除明显重复片段
- 去除无意义语气词，但不改变核心含义
- 修正常见 ASR 断句问题
- 统一时间戳格式

验收标准：

- 清洗后文本仍保留时间戳。
- 原始文本和清洗文本都可导出。
- 对 30 分钟视频清洗后，重复行数量明显减少。
- 清洗过程不得删除连续 20 秒以上的有效内容。

### 4.6 分块处理

将长文本按时间和语义切分成多个块。

建议规则：

- 默认每块 8-15 分钟。
- 优先按自然停顿、字幕段落或主题变化切分。
- 每块保留开始时间和结束时间。

验收标准：

- 60 分钟视频会被切分为多个块。
- 每个块都有明确时间范围。
- 任意单块文本长度不超过配置的 LLM 输入上限。
- 分块结果可在调试页面或日志中查看。

### 4.7 分段总结

对每个文本块生成局部摘要。

每段输出：

- 本段主题
- 关键观点
- 重要例子
- 提到的人名、书名、工具或概念
- 可引用时间戳

验收标准：

- 每个文本块都生成一份结构化摘要。
- 摘要中至少包含 3 个关键观点，除非原文内容不足。
- 关键观点应尽量带时间戳。
- LLM 调用失败时支持重试单个块。

### 4.8 全局总结

整合所有分段摘要，生成完整学习笔记。

默认输出结构：

- 一句话总结
- 3 分钟速读版
- 详细大纲
- 核心观点
- 章节时间线
- 重要概念解释
- 可执行建议
- 值得回看的时间点
- 原始材料质量说明

验收标准：

- 30-90 分钟视频能生成完整总结。
- 全局总结不能只拼接分段摘要，必须做去重和归纳。
- 时间线必须按视频时间顺序排列。
- 输出内容能导出为 Markdown 文件。

### 4.9 导出能力

MVP 阶段优先支持 Markdown。

导出文件包括：

- summary.md：完整总结
- transcript.cleaned.md：清洗后的转写文本
- transcript.raw.md：原始字幕或转写文本
- metadata.json：视频元数据和处理状态

验收标准：

- 每个任务完成后生成独立输出目录。
- Markdown 文件可直接被 Obsidian、Notion、Typora 打开。
- metadata.json 记录输入来源、视频标题、时长、字幕来源、ASR 模型和处理时间。

## 5. 推荐技术架构

### 5.1 模块划分

```text
Input Resolver
  -> Subtitle Fetcher
  -> Audio Extractor
  -> ASR Transcriber
  -> Text Cleaner
  -> Chunker
  -> LLM Summarizer
  -> Exporter
```

### 5.2 建议技术栈

- 后端语言：Python
- 视频/音频处理：yt-dlp、ffmpeg
- YouTube 字幕：youtube-transcript-api
- 本地转写：faster-whisper
- LLM 接入：OpenAI API、兼容 OpenAI 格式的模型服务或本地模型
- 任务队列：MVP 可先用本地同步任务；后续使用 Celery/RQ
- 存储：本地文件系统；后续可接 SQLite/PostgreSQL
- 前端：MVP 可先命令行；后续提供 Web UI

## 6. 开发阶段与验证步骤

### 第 1 阶段：命令行原型

目标：

实现从单个视频链接到 Markdown 总结的最短闭环。

功能：

- 接收一个 YouTube 链接
- 获取字幕
- 清洗文本
- 调用 LLM
- 输出 summary.md

验证：

- 使用一个带字幕的 10 分钟 YouTube 访谈视频测试。
- 运行一条命令即可生成 summary.md。
- summary.md 包含标题、摘要、要点和时间线。

完成标准：

- 不需要人工复制字幕。
- 不需要手动整理文本。
- 处理失败时能看到明确错误信息。

### 第 2 阶段：无字幕视频支持

目标：

支持没有字幕的视频。

功能：

- 下载或提取音频
- 使用 faster-whisper 转写
- 生成带时间戳 transcript
- 继续复用总结流程

验证：

- 使用一个无字幕的 10-20 分钟中文视频测试。
- 系统自动进入 ASR 流程。
- 输出 transcript.raw.md、transcript.cleaned.md 和 summary.md。

完成标准：

- 无字幕视频不需要人工干预即可完成总结。
- 转写结果可追溯到时间戳。

### 第 3 阶段：长视频分块总结

目标：

稳定处理 60-120 分钟课程或讲座。

功能：

- 自动分块
- 分块摘要
- 全局归纳
- 失败块重试

验证：

- 使用一个 60 分钟以上课程视频测试。
- 系统生成 chunk summaries。
- 全局总结包含去重后的结构化笔记。

完成标准：

- 不因文本过长导致 LLM 调用失败。
- 任意单块失败不影响其他块处理。
- 可以重试失败块并重新生成总摘要。

### 第 4 阶段：B站支持

目标：

支持 B站讲座、访谈、课程视频。

功能：

- 解析 B站链接
- 获取标题、分 P、时长
- 尝试获取字幕
- 字幕失败时使用音频转写

验证：

- 使用一个有字幕的 B站视频测试。
- 使用一个无字幕的 B站视频测试。
- 多 P 视频能按分 P 输出或合并输出。

完成标准：

- 单 P 视频可完整处理。
- 多 P 视频至少支持逐 P 处理。
- 平台接口失败时能降级到音频转写。

### 第 5 阶段：批量处理

目标：

支持一次处理多个视频。

功能：

- 输入 URL 列表
- 任务状态记录
- 已完成任务跳过
- 失败任务重试

验证：

- 输入 5 个视频链接。
- 系统依次处理并生成 5 个输出目录。
- 中途失败后重新运行不会重复处理已完成任务。

完成标准：

- 每个任务有独立状态。
- 可以查看成功、失败和处理中数量。

### 第 6 阶段：Web UI

目标：

提供简单可用的图形界面。

功能：

- 输入链接或上传文件
- 显示任务进度
- 查看摘要
- 下载 Markdown
- 查看原始转写文本

验证：

- 用户可以通过浏览器提交视频链接。
- 页面显示下载、转写、总结、导出等阶段状态。
- 任务完成后可以直接预览 summary.md。

完成标准：

- 非技术用户不需要使用命令行。
- 失败状态有明确说明。

## 7. 配置项

MVP 应支持以下配置：

```yaml
asr:
  model: medium
  language: auto
  device: auto

chunking:
  target_minutes: 12
  max_tokens: 6000

summary:
  provider: openai_compatible
  model: configurable
  temperature: 0.2

output:
  format: markdown
  keep_raw_transcript: true
  keep_audio: false
```

验收标准：

- 用户可以修改 ASR 模型大小。
- 用户可以切换 LLM 模型。
- 用户可以设置是否保留原始音频。

## 8. 输出模板

默认 summary.md 模板：

```markdown
# {{video_title}}

## 一句话总结

## 3 分钟速读

## 详细大纲

## 核心观点

## 章节时间线

## 重要概念

## 可执行建议

## 值得回看的时间点

## 材料质量说明
```

验收标准：

- 所有生成的 summary.md 都遵循统一结构。
- 空章节应隐藏或标记为“未识别到相关内容”。

## 9. 错误处理

系统需要覆盖以下错误：

- 链接无效
- 视频不可访问
- 字幕不可用
- 下载失败
- 音频提取失败
- ASR 失败
- LLM 调用失败
- 输出文件写入失败

验收标准：

- 每类错误都有用户可读的错误信息。
- 技术日志中保留详细异常。
- 可恢复错误支持重试。

## 10. 合规与风险

注意事项：

- 遵守 YouTube、B站等平台服务条款。
- 不绕过付费、登录或 DRM 限制。
- 默认仅处理用户有权访问的视频。
- 不默认保存完整视频文件。
- 对课程、付费内容和内部会议资料做好本地隐私保护。

验收标准：

- 文档和 UI 中明确提示用户只处理有权访问的内容。
- 默认不长期保存下载的视频源文件。
- 本地输出目录不包含敏感访问凭据。

## 11. 后续增强方向

- 支持说话人分离：无字幕视频进入 ASR 流程时，可选识别不同说话人并在 transcript 中标注 Speaker 1、Speaker 2 等。
- 支持声音类型辅助推测：可选标注“疑似男声/疑似女声”等信息，但仅作为参考，不作为强准确字段或身份判断依据。
- 支持关键词搜索
- 支持 Notion/Obsidian 导出
- 支持自动生成 Anki 卡片
- 支持课程知识图谱
- 支持关键帧 OCR，用于识别 PPT 标题和屏幕文字
- 支持多模型对比总结
- 支持本地 LLM 私有化部署

## 12. MVP 完成定义

当系统满足以下条件时，可视为 MVP 完成：

- 可处理 YouTube 有字幕视频。
- 可处理无字幕视频并自动转写。
- 可处理 60 分钟以上长视频。
- 可生成结构化 Markdown 总结。
- 可保留原始转写和清洗后文本。
- 每个处理阶段都有明确状态和错误信息。
- 至少通过 3 个真实样例验证：短访谈、长课程、无字幕中文视频。
